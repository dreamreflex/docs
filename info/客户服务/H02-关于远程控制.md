# 关于远程控制

为了更好地向您提供服务，部分项目可能需要在您的本地电脑中部署软件或安装必要的应用程序。为便于远程协助与问题排查，本文将介绍常见的远程控制工具及其下载方式，并补充在无图形界面（GUI）环境下的安全接入方案（如 SSH 密钥登录），以满足不同使用场景的需求。

---

## 一、远程协助工具说明与下载方式

在需要我们进行远程排查或配置时，您可根据实际情况选择以下工具之一进行安装与使用。

### 1. 向日葵远程控制

适用于国内网络环境，部署简单，适合快速建立连接。

* 官网：[https://sunlogin.oray.com/](https://sunlogin.oray.com/)
* 使用方式：下载安装客户端 → 提供识别码与访问密码 → 我方即可协助连接
* 适用场景：Windows / macOS 桌面远程协助

---

### 2. TeamViewer

国际通用远程控制软件，适合跨境或复杂网络环境。

* 官网：[https://www.teamviewer.com/](https://www.teamviewer.com/)
* 使用方式：运行客户端 → 提供 ID 与密码 → 建立连接
* 适用场景：跨地域支持、企业级环境

---

### 3. ToDesk

轻量级远程控制工具，连接速度较快，操作简单。

* 官网：[https://www.todesk.com/](https://www.todesk.com/)
* 使用方式：下载安装 → 提供设备代码与密码
* 适用场景：日常远程协助、低延迟需求场景

---

### 4. RustDesk

开源远程控制软件，支持自建服务器，适合对数据安全有较高要求的用户。

* 官网：[https://rustdesk.com/](https://rustdesk.com/)
* 使用方式：安装客户端 → 输入远程 ID → 验证连接
* 适用场景：开发者、自建环境、内网控制

---

## 二、无图形界面（CLI）环境下的接入方式（SSH）

对于 Linux 服务器或无图形界面的系统，通常不使用远程桌面工具，而采用 SSH（Secure Shell）进行远程访问。为提高安全性，推荐使用“密钥登录”方式替代密码登录。

### 1. 生成 SSH 密钥

在本地终端执行以下命令：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

说明：

* `-t ed25519`：使用更安全高效的加密算法（推荐）
* `-C`：备注信息（可填写邮箱或标识）

执行后会提示：

* 保存路径（默认 `~/.ssh/id_ed25519`）
* 是否设置密码（建议设置，用于增强安全性）

生成完成后，会得到：

* 私钥：`~/.ssh/id_ed25519`（请妥善保管，**不要泄露**）
* 公钥：`~/.ssh/id_ed25519.pub`

---

### 2. 配置服务器授权

将公钥内容追加到服务器：

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出内容，并在服务器上执行：

```bash
mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys
```

将公钥粘贴进去并保存，然后设置权限：

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

---

### 3. 测试 SSH 登录

```bash
ssh username@server_ip
```

若配置成功，将无需输入密码即可登录（或仅需输入密钥密码）。

---

### 4. （可选）提升安全性建议

* 禁用密码登录（仅允许密钥登录）：
  编辑 `/etc/ssh/sshd_config`：

  ```
  PasswordAuthentication no
  ```

  然后重启 SSH 服务：

  ```bash
  systemctl restart ssh
  ```

* 修改默认端口（防止扫描攻击）

* 配置防火墙或 Fail2Ban

---

## 三、说明与建议

* 若您不熟悉操作，可选择图形化远程工具（如向日葵或 ToDesk），由我们协助完成配置
* 若为服务器或生产环境，推荐优先使用 SSH 密钥方式，安全性更高
* 在任何情况下，请**不要向他人泄露私钥或敏感凭据**

如在安装或配置过程中遇到问题，可将错误信息反馈给我们，我们将协助您进一步排查与解决。
